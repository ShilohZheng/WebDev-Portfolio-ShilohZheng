Implemented Accessibility Features

I designed this website with accessibility in mind, following beginner-level WCAG guidelines to help all possible users have a better experience.

  • Semantic HTML5 tags: I used tags like <header>, <nav>, <main>, and <footer> for structure.
    // Code Source: HTML — Helps assistive tech users know what each section means. 
    <main class="container">
  <section>
    <h2>Implemented Accessibility Features</h2>
  </section>
</main>
  • Color contrast: I used dark text on a light background with soft tones.
    // Code Source: CSS — Improves visibility for users with low vision or color blindness. 
    body {
  background-color: #f6f6f6;
  color: #333333;
  font-family: 'Times New Roman', serif;
}
  • Keyboard-navigable buttons and links: All links are standard HTML <a> tags, which are focusable with Tab key.
    // Code Source: HTML & CSS — Makes site usable for people with motor impairments. 
    <nav>
  <a href="index.html">Home</a>
  <a href="portfolio.html">Portfolio</a>
</nav>
  • Readable fonts: I used Times New Roman and adjusted sizes for clarity.
    // Code Source: CSS — Supports users with visual or cognitive disabilities. 
    body {
  font-family: 'Times New Roman', serif;
  font-size: 16px;
}
main.container {
  font-size: 15px;
}
  • Responsive design: I used media queries to adjust layout for different screens.
    // Code Source: CSS (media queries) — Ensures access on phones, tablets, and desktops. 
    @media (max-width: 768px) {
  nav {
    display: flex;
    flex-direction: column;
  }
}
