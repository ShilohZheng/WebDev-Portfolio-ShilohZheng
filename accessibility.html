Implemented Accessibility Features

I designed this website with accessibility in mind, following beginner-level WCAG guidelines to help all possible users have a better experience.

  • Semantic HTML5 tags: In my website, features like <header>, <nav>, <main>, and <footer> help screen readers understand page structure.
    // Code Source: HTML — Helps assistive tech users know what each section means.
  • Color contrast: The color choices of light backgrounds and deep text colors meet the basic readability standards.
    // Code Source: CSS — Improves visibility for users with low vision or color blindness.
  • Keyboard-navigable buttons and links: All interactive elements like nav links and buttons can be used with just a keyboard, as me myself is using Mac without a mouse.
    // Code Source: HTML & CSS — Makes site usable for people with motor impairments.
  • Readable fonts: I use Times New Roman for clarity and formality, and adjust the text sizes for easy reading.
    // Code Source: CSS — Supports users with visual or cognitive disabilities.
  • Responsive design: The layout of my website adapts to different screen sizes for better mobile accessibility.
    // Code Source: CSS (media queries) — Ensures access on phones, tablets, and desktops.
